# 🏠 Homelab Kubernetes — Talos + Terraform + Proxmox

Fully automated Kubernetes cluster using **Talos Linux** on two non-clustered **Proxmox** hosts, provisioned with **Terraform**.

## 🖥️ Hardware

| Host | Hardware | RAM | Role |
|---|---|---|---|
| **pve1** | Intel NUC i3-1115G4 | 64 GB | Main K8s host (3× CP + 1× worker) |
| **pve2** | Intel N100 | 16 GB | Light worker ⚠️ OPNsense lives here |
| **rpi1** | Raspberry Pi | 8 GB | HomeAssistant, Prometheus, Grafana, Pi-hole |

## 🗺️ Cluster Layout

| VM | Role | Host | IP | vCPU | RAM |
|---|---|---|---|---|---|
| talos-cp-1 | Control Plane | pve1 | 192.168.1.101 | 2 | 4 GB |
| talos-cp-2 | Control Plane | pve1 | 192.168.1.102 | 2 | 4 GB |
| talos-cp-3 | Control Plane | pve1 | 192.168.1.103 | 2 | 4 GB |
| talos-worker-1 | Worker | pve1 | 192.168.1.111 | 2 | 20 GB |
| talos-worker-2 | Worker | pve2 | 192.168.1.112 | 2 | 6 GB |
| **VIP** | kube-apiserver | — | 192.168.1.100 | — | — |

## 📦 Stack

| Component | Tool |
|---|---|
| OS | Talos Linux (immutable, no SSH) |
| IaC | Terraform (`bpg/proxmox` + `siderolabs/talos` providers) |
| CNI | Cilium (eBPF, replaces kube-proxy + MetalLB) |
| Storage | Longhorn (2-replica across both workers) |
| TLS | cert-manager |
| Ingress | Cilium Gateway API |
| Observability | Hubble (built into Cilium) |

## 🚀 Quick Start

### Prerequisites

```bash
# Install required tools
brew install terraform talosctl kubectl helm cilium-cli
# or on Linux — see each tool's install docs
```

### 1. Create Proxmox API Token

In Proxmox UI: **Datacenter → Permissions → API Tokens → Add**
- User: `root@pam`
- Token ID: `terraform`
- Privilege Separation: **unchecked**

### 2. Configure environment

```bash
# Copy and edit the env script
cp scripts/setup-env.sh scripts/setup-env.local.sh
# Edit setup-env.local.sh — add your actual API token
source scripts/setup-env.local.sh
```

### 3. Configure variables

```bash
cd terraform/
cp terraform.tfvars.example terraform.tfvars
# Edit terraform.tfvars with your actual IPs
```

Key values to change in `terraform.tfvars`:
- `pve1_endpoint` / `pve2_endpoint` — your Proxmox host IPs
- `cluster_vip` — unused static IP for the K8s API (e.g. `192.168.1.100`)
- `cluster_gateway` — your OPNsense LAN IP
- `cluster_dns` — your Pi-hole IP on rpi1
- `cp_ips` / `worker_pve1_ip` / `worker_pve2_ip` — static IPs for VMs

Also update `talos/patches/controlplane.yaml`:
- Change the `vip.ip` to match your `cluster_vip`

Also update `kubernetes/cilium/values.yaml`:
- Change `k8sServiceHost` to match your `cluster_vip`

Also update `kubernetes/cilium/ip-pool.yaml`:
- Change the IP range to a free block on your LAN

### 4. Deploy

```bash
cd terraform/
terraform init
terraform plan
terraform apply
```

This single command:
- Downloads Talos images to both Proxmox hosts
- Creates all 5 VMs
- Generates machine configs with patches
- Applies configs via Talos API (no SSH!)
- Bootstraps etcd
- Waits for cluster health
- Writes `output/talosconfig` and `output/kubeconfig`

### 5. Verify cluster

```bash
export TALOSCONFIG=$(pwd)/../output/talosconfig
export KUBECONFIG=$(pwd)/../output/kubeconfig

kubectl get nodes
talosctl health
```

### 6. Post-install (CNI + Storage)

```bash
cd ..
bash scripts/post-install.sh
```

Installs: Cilium, Longhorn, cert-manager

## 📁 Project Structure

```
homelab-k8s/
├── terraform/
│   ├── providers.tf          # bpg/proxmox (pve1 + pve2) + siderolabs/talos
│   ├── variables.tf          # All input variables
│   ├── terraform.tfvars.example  # Template — copy to terraform.tfvars
│   ├── image.tf              # Downloads Talos image to both PVE hosts
│   ├── vms.tf                # Creates all 5 VMs
│   ├── talos.tf              # Machine configs, bootstrap, health check
│   └── outputs.tf            # Writes kubeconfig + talosconfig to output/
├── talos/
│   ├── image/schematic.yaml  # Talos extensions (qemu-guest-agent, iscsi-tools)
│   └── patches/
│       ├── controlplane.yaml # VIP, disable kube-proxy, no CNI
│       └── worker.yaml       # Longhorn mounts, sysctls
├── kubernetes/
│   └── cilium/
│       ├── values.yaml       # Helm values for Cilium
│       └── ip-pool.yaml      # LoadBalancer IP pool + L2 announcement policy
├── scripts/
│   ├── setup-env.sh          # Export API tokens + config paths
│   ├── post-install.sh       # Install Cilium, Longhorn, cert-manager
│   └── upgrade-talos.sh      # Rolling Talos OS upgrade
└── output/                   # Generated by terraform apply (gitignored)
    ├── talosconfig           # talosctl credentials
    └── kubeconfig            # kubectl credentials
```

## ⚠️ Important Notes

1. **pve2 safety** — `talos-worker-2` RAM is capped at 6 GB with no ballooning. OPNsense must always have headroom. Never increase this without checking total pve2 RAM usage.

2. **Terraform state** — contains your cluster PKI. Back it up! Consider a remote state backend (Terraform Cloud, MinIO + S3 backend).

3. **VIP IP** — the `cluster_vip` must be the same in three places: `terraform.tfvars`, `talos/patches/controlplane.yaml`, and `kubernetes/cilium/values.yaml`.

4. **Static IPs** — all VM IPs must be outside your DHCP range. Configure static leases in OPNsense to ensure they don't change.

5. **Pi-hole DNS** — all K8s nodes use Pi-hole as DNS. If rpi1 is down, nodes lose DNS. Consider adding a secondary DNS (e.g. `1.1.1.1`) as fallback in `terraform.tfvars cluster_dns`.

## 🔧 Useful Commands

```bash
# Cluster health
talosctl health

# Node details
talosctl get members

# Logs from a node (replaces journalctl)
talosctl logs kubelet --nodes 192.168.1.101

# Dashboard (live node stats)
talosctl dashboard --nodes 192.168.1.101

# Reboot a node safely
kubectl drain talos-worker-2 --ignore-daemonsets --delete-emptydir-data
talosctl reboot --nodes 192.168.1.112
kubectl uncordon talos-worker-2

# Upgrade Talos OS (rolling)
bash scripts/upgrade-talos.sh v1.10.0 <schematic-id>

# Upgrade Kubernetes version
talosctl upgrade-k8s --to 1.33.0

# Longhorn UI
kubectl port-forward -n longhorn-system svc/longhorn-frontend 8080:80

# Hubble network flow UI
kubectl port-forward -n kube-system svc/hubble-ui 12000:80
```
